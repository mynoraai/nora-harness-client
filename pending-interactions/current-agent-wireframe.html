Hardware Harness / Mobile UI Contract

Mobile Current Agent Interaction Wireframe

First-pass wireframe: reorganizes the four bottom tabs, Agents, Marketplace, Device, and Me, and maps current primary-agent binding state into mobile page states, header state, error states, and recovery paths.

Artifact Static HTML wireframe, not final visual design.
Scope Mobile interaction, user journeys, and state contracts.
Phone Frame 390 × 844 design pixels, rendered at 75% scale.
References ADR 008, shared contracts, existing screenshots.

User Journeys This Version Must Cover

1

Sign In, Then Enter Main App

ONB-001 only handles sign-in. After successful sign-in, the user enters the Agents home and persistent banners continue setup guidance.

2

Pair Device From Main App

The device banner and Device tab both enter pairing. Pairing removes only the device banner.

3

Purchase And Install Agent

Marketplace list opens Agent detail. Purchase changes the primary action to Install, then Agents shows the purchased or installed Agent.

4

Message Agent From Agents

Agents is the conversation list. Tapping an owned Agent opens chat and sending completes this journey.

5

Device Hardware View

The Device tab shows overview, health, and logs; it does not own onboarding reset.

6

Me Account Management

Me owns account information and sign-out. Sign-out returns to ONB-001.

7

Me Device Settings

Me owns device settings, pairing another device, and rename. Device removal is out of this pass.

Core State Principles

Signed Out

Cannot Pair Device

Sign-in is the prerequisite for pairing and purchase/adoption ownership. After sign-in, enter the main app directly.

Device Not Paired

Show Device Banner

This banner disappears after successful pairing. It does not depend on whether the user has an Agent.

No Agent

Show Agent Banner

This banner disappears once the mobile app has purchased or adopted any Agent.

Purchased / Adopted

Phone Chat Enabled

The Agent instance keeps running on the phone. When not installed to hardware, device action entry points are not shown.

Not Installed To Hardware

Hardware State Lives In Header

The header toolbar shows Not installed. Chat stays clean, with no extra toolbar below the header.

Installed To Hardware

Binding State Reaches Device Side

After Install to device, Device and Chat header show current binding. Chat remains the same conversation.
