Journey notes

User outcome, Constraints, Evidence gaps
User outcome

Users start from the real Chat shell, select or confirm a model, write a prompt, and keep recovery controls in the same conversation.

Constraints

Only states visible in or directly implied by the current screenshots are drawn; the wireframe avoids redesigning the Chat surface.

Evidence gaps

Need a current screenshot for an actual streamed answer/error turn before finalizing message-row details.
