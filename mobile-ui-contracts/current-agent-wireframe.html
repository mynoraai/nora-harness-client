Placement And Intent

为什么放这里

这个文件属于手机端交互契约，和现有 onboarding、chat、agents、device 截图同源，所以放在 mobile-ui-contracts 目录，方便后续实现和验收引用。

不是 sprint 附件

sprint 目录适合任务节奏和当周清单；这里表达的是跨 sprint 仍然有效的 UI flow 和状态契约。

技术关联

页面所有可发送、可切换、可卸载的状态，都必须从 desired/reported binding、generation ack 和 runtime readiness 推导，而不是只看 install 是否存在。

User Journey

1

登录

onboarding 保留账号登录，确保购买/认领、同步和设备归属有身份基础。

2

蓝牙配对

保留蓝牙扫描、验证码确认和本机 pairing，让 App 知道当前设备是谁。

3

可跳过的 Agent 选择

onboarding 可以推荐 Agent，但必须允许跳过并进入 paired + unbound 主界面。

4

安装并设为当前

如果用户选择 Agent，则创建 desired primary install，进入 pending。

5

设备确认

Firmware 拉取或收到 binding，应用后回报 matching generation。

6

进入主界面

已确认 binding 时聊天解锁；跳过 Agent 时进入未绑定态并引导后续选择。

Binding States

unbound

已配对，没有 current Agent

Device 可以管理，Chat 入口保留但输入禁用，引导用户去 Marketplace 或 Device 绑定。

binding / pending

desired 已写入，等待 reported

显示进度、禁止重复发送和重复切换；允许取消或查看设备状态。

bound

desired/reported generation 匹配

当前 Agent 成为 Agents tab 主体验；Chat 输入、语音、历史会话正常启用。

failed

安装或绑定失败

不要静默进入 Chat。展示失败原因、重试、换 Agent、查看 Device 诊断。

stale

本地投影旧于设备状态

Chat 降级为只读或禁用输入，触发 refresh binding，避免对旧 runtime 发消息。

reprovision_required

需要重新配置设备

Device tab 提供恢复入口，重新确认网络、pairing 和 desired binding。
