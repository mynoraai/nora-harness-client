Blocking PM decision

Pre-send readiness model changes user friction.

blocking
Should runtime readiness failures block Send before the user composes, or appear only after the first failed send/retry?

Blocking earlier is safer. Blocking later keeps exploration fast but can waste a written task.

Error and recovery states
