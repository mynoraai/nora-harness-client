Workspace Creation Detail

The required local project step.

required
  • 1Code empty state invites the user to create a workspace.
  • 2Native folder picker returns a local directory or cancel.
  • 3Name auto-fills from the folder and remains editable.
  • 4Create stays disabled until a directory is selected.
  • 5Workspace becomes active and owns the first code conversation.

Runtime Choice Detail

Runtime is flexible, but first draft has a default.

choice
DefaultNew Code draft starts on Claude Code.
ChoicesOpenClaw, Claude Code, and Codex are selectable in the composer.
Lock ruleAfter a native runtime identity exists, casual switching is blocked.

Engineering Readiness

PM Decisions

Choices this artifact is meant to unblock.

PM review
Blocking: Should Code mode require workspace selection before first send, or allow task-first entry?
Decision: Should Claude Code remain the default runtime, or should first-time users choose explicitly?
Decision: Should the draft conversation be visible in the workspace group before first send?

Discussion Boundaries

Keep the review on first Code adoption.

scope
DiscussWorkspace-first flow, runtime default, visible draft state, readiness cards, recovery copy.
Do not discussFile tree, diff, terminal, Agent Authoring, device panels, provider setup.

Workspace Recovery

Folder failures should not create dead state.

important
Picker canceledKeep dialog open with Create disabled, or return to empty state if user closes.
No directoryBefore send, ask user to choose a working directory.
Bad folderShow folder unavailable state without deleting draft work.

Runtime Recovery

First-send failures need clear ownership.

recovery
Model unavailablePrompt user to select an enabled model before sending.
Runtime missingShow install or configure guidance tied to selected runtime.
Auth requiredKeep task text and show retry after sign-in/auth fix.

Definition Of Success

What "started a Code session" means.

success
  • 1User has an active local workspace.
  • 2A New Conversation tab and composer are visible.
  • 3Runtime and model are selected or clearly blocked.
  • 4First task is handed to the selected code runtime exactly once.
