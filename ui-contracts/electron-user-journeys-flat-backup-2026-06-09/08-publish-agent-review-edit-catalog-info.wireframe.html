Journey 08 - Publish Agent, Review, Edit Catalog Info

Publishing turns a drafted agent into a reviewable package with catalog details, review status, and post-submit edits.

Status: needs verificationVerification point: Verify the publish checkpoints and which catalog edits remain allowed during review.

Journey notes

User outcome, constraints, evidence gaps
User outcome

Review status becomes understandable for authors and reviewers. Catalog edits are separated from package version changes.

Constraints

More publish checkpoints add friction. Review terminology must match backend and marketplace language.

Evidence gaps

Can catalog metadata be edited after approval but before the agent becomes live?
