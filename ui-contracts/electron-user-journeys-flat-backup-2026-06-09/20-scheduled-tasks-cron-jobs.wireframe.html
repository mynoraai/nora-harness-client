Journey 20 - Scheduled Tasks / Cron Jobs Priority Journey

Scheduled Tasks should become a first-class way to create one-shot reminders, recurring isolated jobs, conversation-derived drafts, and job management without confusing them with heartbeat or background sync.

Status: not current surfaceVerification point: Verify the first-class Scheduled Tasks model, including entry point, task types, required fields, and isolation language.

Jump list

Main path

5 frames

Entry point

A Scheduled Tasks item appears in the main app navigation or workspace tools, not hidden in advanced settings.

  • Proposed
  • Badge for due/failed

Task hub

Hub lists upcoming, running, failed, paused, and completed tasks with filters and quick create.

  • Upcoming first
  • Run history access

Create task

User chooses One-shot reminder, Recurring isolated job, or Draft from conversation.

  • Type cards
  • Clear differences

Review and save

Task summary shows schedule, owner context, permissions, notification behavior, and enabled state.

  • Save disabled until valid
  • Test run optional

Monitor result

Task detail shows next run, last run, transcript/output, failure recovery, pause, duplicate, and delete.

  • Audit trail
  • Manual run

Task types

3 frames

One-shot reminder

User enters title, due time, optional context, and notification channel; completion clears it from upcoming.

  • Date/time required
  • No recurring fields

Recurring isolated job

User defines recurrence, task prompt, workspace or agent context, permission policy, and failure handling.

  • Runs isolated
  • No open chat required

Create from conversation

Assistant proposes a scheduled task draft from chat; user must review schedule and permissions before save.

  • Draft only
  • User confirms

Required fields

3 frames

Core fields

Every task needs name, type, schedule, enabled state, owner context, notification preference, and failure behavior.

  • Validation list

Recurring fields

Recurring jobs need recurrence rule, timezone, start/end or no-end choice, missed-run behavior, and max runtime.

  • Timezone visible

Execution fields

Jobs that act on files or devices need workspace, approval policy, and safe retry language.

  • Permission clarity

Concept split

3 frames

Not heartbeat

Heartbeat is an agent behavior concept; Scheduled Tasks are user-created jobs with explicit schedule and audit trail.

  • Separate labels

Not background sync

Background maintenance remains invisible unless it needs attention; user tasks are visible and manageable.

  • No mixed list

Not hidden advanced cron

Advanced cron strings can be an expert input later, but the default creation path uses human schedule controls.

  • Human schedule first

  • Verify

Journey notes

User outcome, constraints, evidence gaps
User outcome

Users can trust tasks to run without keeping a chat open. Reviewers can separate reminders, automations, and background behaviors cleanly.

Constraints

First-class navigation adds product surface area. Recurring jobs need careful safety and failure language. Conversation-derived drafts require strong confirmation to avoid accidental automation.

Evidence gaps

Should a task belong to the whole app, a workspace, an agent, or a conversation by default? Should expert cron expression entry ship in the first version or be deferred behind advanced mode?
