Journey notes

User outcome, constraints, evidence gaps
User outcome

Model readiness is visible before users start a session. Default model problems have a clear recovery path.

Constraints

Provider pages can become dense with advanced options. Cost/capability hints may need ongoing maintenance.

Evidence gaps

Should default model selection warn when the chosen model is weak for code or hardware tasks?
