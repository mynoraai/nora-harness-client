# Electron User Journeys Index

## Purpose

This package tracks the 20 numbered Electron desktop user journeys as factual interaction records. The HTML files are reviewable wireframes; the Markdown files explain the current journey inventory and identify which flows still need visual/product verification.

This is not a proposal document. Do not treat verification points in the HTML as new product decisions. They are places where the existing journey needs to be checked against current screenshots, docs, or product behavior.

## Source Documents

- [Journey overview](user-journeys.md)
- [Journey audit](journey-audit.md)

## Current State

- Journeys 01 and 02 have been updated toward product-shaped wireframes using current onboarding/startup references.
- Journeys 03-10, 12-16, 18, and 19 still use summary-card wireframes and need visual verification before they are treated as final interaction records.
- Journeys 11 and 17 are removal placeholders, not active user journeys.
- Journey 20 is a priority concept / not-current surface until product evidence confirms it should be represented as a current UI.

## Maintenance Notes

- Keep HTML and Markdown synchronized. If a journey is marked removed or not current in Markdown, the HTML should not draw it as an active UI.
- Keep verification points factual. They should ask what still needs evidence, not ask reviewers to choose a new product direction.
- When current screenshots exist, use them before changing a journey's product frame.
