Status language

Pills are the backend review states. Visibility is a separate overlay, not a status.

DraftLocal in the IDE; not submitted. Pending reviewSubmitted; in the review queue. In reviewA reviewer has opened it. ApprovedPassed review, not live yet — awaiting the author's Publish to production. PublishedLive to the public (liveVersionId points here). RejectedTerminal; create a new version. RevokedWas published, then removed.
Visibility (derived, not a status). ● live = liveVersionId points here (published is not live). ● latest for device = newest published firmware for a channel + device model.

Review state machine · per version

Adds one state — approved (passed review, not live). Approve and release are separate steps.

draft · local — Submit for review → pending_review — open → in_review — approve → approved — Publish to production → published · live
in_review — reject → rejected  ·  published — revoke → revoked  ·  Publish to production sets liveVersionId
draft = local (not submitted). Test build sees pending_review / in_review / approved; public build sees published (liveVersionId). approved = passed review, awaiting the author's Publish to production.
