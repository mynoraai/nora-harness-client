Status language

Pills are the backend review states. Visibility is a separate overlay, not a status.

DraftOn test — testers install it; not yet in production review. Pending reviewRequested production — awaiting a reviewer. In reviewA reviewer has opened it. PublishedApproved and eligible. RejectedTerminal; create a new version. RevokedWas published, then removed.
Visibility (derived, not a status). ● live = liveVersionId points here (published is not live). ● latest for device = newest published firmware for a channel + device model.

Review state machine · per version

No new states. draft is on-test; Request production is the draft → pending_review submit (today auto at create).

draft · on test — Request production → pending_review — open → in_review — approve → published · live
in_review — reject → rejected  ·  published — revoke → revoked
Test track has no review (draft — testers install directly). The single production gate is pending_review → in_review → published, entered by Request production. Review off: draft → published directly, still explicit.
