# Chat Contracts

Source rows: `CHAT-01` through `CHAT-08`

## What This Covers

This folder covers what users can do in Chat mode: create and choose sessions, read the message list, type in the composer, attach files, use slash commands, pick a model, choose thinking level, send or stop a run, reconnect to an active run, and recover from visible errors.

Code-mode chat tabs reuse some components, but Code-specific tab and workspace behavior belongs to `code-ide/`.

## First-Time Reading Order

1. Start with `chat-runtime.md#purpose` for the user-visible Chat flow.
2. Read `chat-runtime.md#core-responsibilities` to understand the split between sidebar, message list, composer, and model/thinking controls.
3. Follow `chat-runtime.md#step-by-step-reader-guide` before reading the visible-controls matrix.
4. Read `client-state-machine.md#purpose` only after the visible flow is clear; that file explains how streaming events become message parts.
5. Use this README table as the row index for `CHAT-*` evidence and test mapping.

## Detail Files

- `chat-runtime.md` covers the Chat layout, visible controls, history load, composer states, send/stop/reconnect behavior, and message rendering.
- `client-state-machine.md` explains how `useChat`, `DualStreamHandler`, snapshot reconciliation, slash-command output, draft session saving, and stop-reason normalization make the visible message list change.

## Transport And State Dependencies

Detailed Chat files should use `provider-gateway-boundaries/transport-and-events.md` and `provider-gateway-boundaries/reference-surfaces.md` when documenting:

- `chat.send`
- `chat.abort`
- `chat.history`
- stream reconnect
- message part normalization
- provider-specific model grouping

Do not duplicate the gateway method trace inside every Chat document.
