# Main Window, Sidebar, Account, And System Routes Contracts

Source rows: `MAIN-01` through `MAIN-06`

## What This Covers

This folder covers the shared Electron frame and OS entry routes around Chat, Code, and Settings: returning-user boot, layout selection, Chat/Code mode switching, sidebar controls, account footer, tray actions, gateway banner, update toast, and pairing-notification click routing.

Do not put detailed Chat or Code behavior here. This folder only covers the shared route into those screens.

Notification scope: this folder covers desktop notification clicks that carry a pairing-channel route. Onboarding notification permission belongs to `onboarding/`; the Settings Permissions tab's notification settings card belongs to `settings/`.

## First-Time Reading Order

1. Start with `main-window-runtime.md#purpose` to understand the shared main-window boundary.
2. Read `main-window-runtime.md#core-responsibilities` to separate the renderer frame, sidebar, account footer, tray, and pairing-notification route ownership.
3. Follow `main-window-runtime.md#step-by-step-reader-guide` before reading individual `MAIN-*` rows.
4. Use this README table as the row index for `MAIN-*` evidence and test mapping.
