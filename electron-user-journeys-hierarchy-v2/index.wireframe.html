Electron User Journeys Hierarchy V2

Redrawn from user-journey-inventory.md, references, and NoraHarness live observation. Each page records entry, user behavior, visible client state changes, and end state.

01. First Launch / Onboarding

02. Main App Startup / Recovery State

After the user reopens NoraHarness, the client restores the previous Chat or Code mode, account state, workspace, session tab, right panel, and Gateway availability.

implemented journey recordlive NoraHarness observation03

03. Login / Account Authentication

The user enters login from the sidebar or a pre-publish gate. After browser authentication, the user returns to NoraHarness and the account area shows identity and Sign out.

implemented journey recordauthentication reference + live app observation04

04. Chat Mode

In Chat mode, the user selects a conversation, reviews the empty-conversation guidance, sets runtime/model/thinking, sends a message, and handles running state.

implemented journey recordlive NoraHarness observation05

05. Code Mode / Workspace

The user switches to Code, creates a workspace, enters a workspace-scoped Code session, and sends the first task.

implemented journey recordworkspace/code reference + live app observation06

06. Runtime / Model / Permission

The user selects runtime, model, and thinking in the composer; while running, the user may encounter locked runtime, provider readiness, or conversation-style approval messages.

implemented journey recordlive app observation + approval screenshot reference07

07. Code Workbench Panels

The right and bottom areas of a Code session provide All Files, Changes, Device, Preview, Terminal, and Device Log.

implemented journey recordlive NoraHarness observation08

08. Agent Authoring

The user enters agent authoring, edits Identity/Soul/Style/Tools/Heartbeat/Bootstrap/Skills/Capabilities, and enters publishing from Publish.

implemented journey recordpublish reference + app information reference09

09. Publish Center / Catalog Review / Firmware

The user enters the unified publish center from Agent Authoring, submits app/agent package review, adds app info/media, and enters firmware publish from the same center.

implemented journey recordlatest publish flow reference + app observation10

10. Device / PlatformIO / Logs

The user checks Device Stream, device connection state, and Device Log in the Code workbench, and starts USB Monitor, filters logs, pauses scrolling, or refreshes when needed.

implemented journey recordlive NoraHarness observation11

11. LVGL Preview

The user checks LVGL preview in the Code workbench Preview tab; missing configuration shows disabled start, while available configuration shows the live view, build/capture, and log.

implemented journey recordpreview reference + live app observation12

12. Settings - General

The user enters Settings General from the sidebar, checks Gateway state, restarts Gateway, edits proxy, toggles feature switches, and changes default thinking.

13. Settings - Workspace Files

The user enters Settings Workspace, selects SOUL/IDENTITY/USER/AGENTS/TOOLS/HEARTBEAT/MEMORY, edits markdown content, and saves.

14. Settings - Providers

The user views default model and provider list in Settings Providers, edits provider API key/base URL/enabled models, and saves.
